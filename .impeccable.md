# Design Context

## Target Audience
Developers running multi-agent orchestration pipelines. Sitting at a desk, monitoring long-running builds. Need to see progress, intervene when agents stall, and debug agent behavior.

## Use Cases
- Confidence that progress is being made (sprint progress, phase transitions)
- Debugging agent behavior (full output logs, contract diffs, eval reports)
- Intervention when things go wrong (stall detection, rollback visibility)

## Brand Personality / Tone
Mission control meets developer terminal. Serious, operational, data-dense but not cluttered. The feel of a NASA flight director's console crossed with a well-configured tmux session. Information-rich, low-chrome, high-signal. Monochrome with selective color for status and alerts.
